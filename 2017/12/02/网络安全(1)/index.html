vpn

虚拟的专有网络，
可以对数据进行加密和完整性保护，也可以不加密，不保护

  1. vpn分类

    1. osi7层
      1. 应用层 ：app 开源vpn（openvpn）
      2. 会话层，表示层 ：没有vpn
      3. 传输层 ：ssl vpn（安全 socket（套接字/端口））
      4. 网络层 ：ipsec vpn（运用度很广）mpls vpn（2.5层）
      5. 数据链路层 :pptp（点到点隧道协议）,l2tp（二层隧道协议）—-逐渐被淘汰（用于保护操作系统数据传输过程）

    2. 应用

      1. L2L vpn；site to site vpn（局域网到局域网，站点到站点） ：

        ipsec vpn—->DM vpn(基于ipsec，为了取代它，思科私有)【ccie考试必考】

        get vpn （通用的 封装加密 隧道）对mpls vpn的安全加固（思科私有）

      2. remote vpn

        ssl vpn（保护邮件）

        pptp，l2tp ez-vpn（思科私有）—->anyconnect vpn（最新）能适配终端更多（ios/Android…）

    3. 明文数据挑战
      1. 对等体认证
      2. 完整性
      3. 加密
      4. 抵赖—-数字签名

    4. vpn组建

      1. key 密钥

        1. 认证的过程中 key 就是口令
        2. 计算过程中 key是工具，变量

        3. 加密算法key

          1. 对称key ：加密解密同一把key，优点：快，高效，加密后的数据以及不会膨胀太多

          2. 非对称key ：

            1. 公key

            2. 私key

              拿到公私钥互相算不出对方，但可以互相解密

            3. 一般情况下，加密公私钥如何分配

              公钥互相交换，公钥加密数据，私钥解密

            4. 数字签名

          对称密钥加密算法保护数据，非对称密钥加密算法保护过程

    5. 对称，非对称算法

      1. 对称的 ：des；3des；aes
      2. 非对称 ：dsa；rsa

    6. 数据完整性保护算法

      1. md5（消息摘要算法）算完128bit（描述一个算法的强壮程度是bit数）
      2. sha（算完256bit，要去代md5）

    7. key的产生

      dh算法 ，两个随机数算出相同的数，且不可逆（魔方算法）

    8. 启源认证/对等体认证

      1. 预共享口令认证
      2. 数字证书 rsa/signature
      3. 加密随机数（淘汰，傻逼）

    9. lsakmp与ike2个阶段

      1. lsakmp互联网安全关联密钥管理协议
        1. sa安全关联—-连接和过程已经被保护了

      2. ike互联网密钥交换（被包含在isakmp）

        a----------------b
1. ike阶段一
    1. （1.2）互发vpn保护策略（方式一样）
    2. （3.4）互发认证消息
    3. （5.6）互发保护vpn的key
2. ike阶段二
    1. （7.8.9）利用阶段一的key算出保护数据的key，确认加密用户数据的算法和完整性的算法

    10. vpn头部

      1. 隧道模式 tunnel mode
      2. 传输模式 transport mode
      3. ah，esp
        1. ah认证头：加密数据只保证认证和完整性
        2. ESP：封装安全静荷（静荷就是中心数据）：加密数据保证认证和完整性

ipsec vpn的配置

  1. 确认路由器ios支持ipsec vpn

    show version //查看版本
K9的含义代表支持128bit以上的加密算法

  2. 完成ipsec阶段一保护策略的设置

    crypto isakmp policy <策略编号>
authentication pre-share //开启预共享口令认证
encrption <des/3des/aes> //设置加密vpn数据流量方式
hash <md5/sha> //设置vpn流量完整性保护方式
group <1/2/5>  //设置dh组算法级别

  3. 设置对等体地址，并设置预共享口令

    crypto iskmp key <口令> address <对端vpn网关公网地址>

  4. 设置ike阶段二 ，用户数据保护方式和数据封装头部方式和vpn方式

    crypto ipsec transet <转换集名称> esp-<des/3des/aes> esp-<md5/sha>-hmac
mode <tunnel/transport>
  5. 编写扩展acl，利用acl匹配需要被vpn保护的流量

  6. 配置加密映射，将对等体，感兴趣流和阶段二用户保护方式结合

    crypto map <机密映射名字> <加密映射编号> ipsec-isakmp
set peer <对端vpn网关公网地址>
set transet <转换集名字>
match address <acl编号>

  7. 加密映射套用在接口上

    crypto map 只对接口发送的流量生效

    interface <intf-id>
crypto map <加密映射名字>
  8. 查看
    show crypto engine connection active  //查看vpn是否机密解密流量
show access-list  //查看acl有没有被击中

ipsec vpn与nat同时存在的问题

nat会先于vpn抓取到流量，所以nat的转换工作可能会导致vpn无法识别该被加密的流量

  1. nat bypass ：nat绕行

    改acl，使通过vpn到达对端的流量不走nat

ipsec vpn实施方式

  1. 隧道ipsec vpn

    先建立gre隧道，利用隧道完成路由可达，再利用ipsec vpn对隧道保护

  2. 设备出口直接使用ipsec vpn

    设备出口流量会被ipsec vpn识别，判断是否需要加密保护

ipsec vpn的缺陷

  1. 对端设备必须公网地址固定
  2. 如果需要全互联IPSEC VPN保护，配置特别复杂
  3. 无法实现分支之间无需IPSEC VPN配置就能建立IPSEC VPN连接

pbr结合ipsec vpn的选路设置

  1. pbr：基于策略的路由

    pbr利用acl匹配出源于不同网络去向不同网络的流量，利用route-map强行设置下一跳

    pbr只能对接口接收的流量生效

  2. ip sla ： ip服务质量

    ip sla 实际上起到探针的作用，能检测连通性

  3. ip track：ip对象跟踪

    对象跟踪可以跟踪特定的目标，并且引用

    ip sla <id> //定义了特定编号的sla探针
icmp-echo <目标ip> source-ip <源ip>  //以特定源去ping特定目的地
ip sla schedule <id> life froever strat-time now  //设定特定编号的sla探针生效多久，什么时候开始
track <对象跟踪的编号> rtr <sla 探针编号> reachability  //定义了特定的对象编号，跟踪特定sla可用性

DM-vpn

动态多点vpn技术 ，思科私有

  1. dm-vpn组件

    1. m-gre 多点隧道

      传统隧道必须指明目的地地址，多点隧道不需要指定

    2. 物理地址和逻辑地址

      物理地址是接口所获得的公网地址

      逻辑地址是管理员配置给m-gre所用（m-gre隧道ip通常是私有地址）

    3. NHRP和NHS

      1. NHRP 下一跳解析协议

        通过nhrp将逻辑地址对应的物理地址解析出来

      2. NHS 下一跳服务器

        所有分支都必须致命谁是下一跳服务器，分支将自己的逻辑地址和所获得的动态物理地址回报给nhs

    4. dm-vpn中所需要的底层协议

      该协议是帮助隧道建立邻居关系，帮助总部和分支交换路由信息

    5. ipsec

      dm vpn最终利用ipsec保护隧道传递数据

  2. dm-vpn的是哪个阶段

    1. 阶段一：所有的分支流量必须要经过hub传递
    2. 阶段二：所有的分支流量只有在第一次的时候通过hub传递，后面就全部都是相互直接传递
    3. 阶段三：hub可以有多个，进行备份

  3. dm-vpn的配置

    1. 在hub上完成隧道的创建和配置

      interface tunnel <id>
ip address <逻辑地址> <掩码>
tunnel source <物理地址>
tunnel key <口令>    //口令不一定要配置，增加安全性
tunnel mode multipoint     //多点模式
    2. 完成spoke的隧道配置

    3. 完成hub上nhrp的配置

      ip nhrp network-id <id>   //设置网络id（类似域名）
ip nhrp authentication <id> //设置认证口令
ip nhrp map multicast dynamic //允许多播流量在m-gre中传递-----底层协议路由更新在m-gre隧道中以多播形式发送

    4. 完成spoke端nhrp配置

      ip nhrp network-id <id>   //设置网络id（类似域名）
ip nhrp authentication <id> //设置认证口令
ip nhrp <逻辑地址>  //指定nhs服务器，该地址是hub的隧道地址
ip nhrp map <hub逻辑地址> <hub物理地址>  //完成nhs服务器逻辑地址和物理地址的关联
ip nhrp map multicast <物理地址>  //spoke向hub发送多播流
    5. 启用底层协议（eigrp）

    6. 完成ipsec的配置和引用

      此处的ipsec不需要写明对等体地址，也不需要编写感兴趣流

      crypto isakmp key xxx address 0.0.0.0 0.0.0.0  //对等体地址是动态的，无法已知

crypto ipsec transform-set xxx esp-des esp-md5-hmac 
mode transport //将模式改为传输模式

crypto ipsec profile xxx
set transform-set xxx  //创建ipsec配置文件，指定转换集

interface tunnel <id>
tunnel protection ipsec profile xxx //隧道流量用ipsec保护

    7. 查看

      show ip nhrp //在hub端用，看spoke是否通过nhrp告诉我spoke端的逻辑ip和物理ip

  1. dm-vpn阶段一

    正常情况下dm-vpn阶段1隧道参与eigrp后、spoke之间不学习路由，因为eigrp的水平分割机制

    dm-vpn阶段一，hub隧道接口下务必关闭水平分割

    interface tunnel <id>
no ip split-horizon eigrp xxx //关闭eirp水平分割

  2. dm-vpn阶段二

    1. 实现方法一：hub上先关闭水平分割，再关闭next-hop-self—–hub不会将自己设置为下一跳了

      interface tunnel <id>
no ip next-hop-self eigrp xxx

    2. 实现方法二：hub上隧道接口下配置手动汇总，分支和核心去配置nhrp的捷径功能

      hub端手动汇总可以减少spoke端路由的学习

      spoke端隧道接口下配置：

      ip nhrp shortcut //配置捷径

      hub端隧道接口下配置：

      ip nhrp redirect //配置重定向

  3. dm-vpn阶段三

    多中心解决方案

    spoke端隧道接口下

    ip nhrp nhs <主nsh服务器地址> <备份nsh服务器地址>
ip nhrp map <主nsh服务器逻辑地址> <主nsh服务器物理地址>
ip nhrp map <备份nsh服务器逻辑地址> <备份nsh服务器物理地址>
ip nhrp map multicast <主nsh服务器物理地址>
ip nhrp map multicast <备份nsh服务器物理地址>

    hub端接口下:两个中心相互映射

    ip nhrp <对端逻辑地址> <物理地址>
ip nhrp

  4. 补充知识

    不同路由协议更新交换方式

    1. rip eigrp路由更新方式一样 ：目标网络，更新源，下一跳
    2. ospf isis的路由更新方式一样 ：传递节点信息，知道网络中有哪些设备

防火墙

  1. 防火墙的定义和分类

    1. 软件防火墙

      侦测流量，发现系统病毒和漏洞

    2. 网络硬件防火墙
      1. nat：隐藏内网地址/地址转换
      2. 病毒侦测，vpn建立，虚拟防火墙，防火墙的热备份
    3. waf ： web应用防火墙

  2. juniper，cisco

    juniper 收购了 诺基亚chekpoint并入了netscreen产品线

    cisco pix第一代 ，现在迭代成asa

    sdn安全自防御网络 sdn软件定义的网络

  3. asa：安全自适应算法

    asa5000

    1. asa上的nat control

      1. asa的接口类型
        1. inside（可信任网络）（security-level 100）
        2. outside（非信任网络）（security-level 0）
        3. dmz（非军事区域，停火区域）（security-level 0）【没卵用】
      2. asa的接口拥有安全级别 ：0-100
      3. asa接口访问规则：
        1. 高安全级别接收流量穿越防火墙从低安全级别出去必须nat
        2. 防火墙的nat不光转换地址，还会转换例如：端口号，回话id，防火墙把nat叫做有状态的转换
        3. pix上可以关闭该功能，流量从高到低可以不需要nat，asa不让关
        4. 低安全级别接收流量，如果流量不是返回流量，也不是被acl允许的，那么是无法穿越防火墙进入inside。
        5. 安全级别相同的接口默认是不允许相互传递流量

    2. 防火墙接口的类型指定，安全级别指定，包括nat的配置

      show interface ip brief //查看接口信息
show route //查看路由表

interface <id>
nameif <inside/outside/xxx> //给接口命令并且指定类型
security-level <0-100>
ip address xxxx xxxx
no sh
接口不命名，接口集火也无法传递数据
//所有cisco防火墙特权模式默认密码为 回车

      接口名字如果是inside/outside那么携带默认安全级别，否则都是0

    3. 防火墙配置静态路由

      route 出口（nameif的名字） 目标网络 目标网络掩码 下一跳

    4. pix和asa配置nat转换

      1. pix nat配置 

        nat（inside）x 被允许做转换的内网网段 子网掩码   //括号里为接口名
global（outside）x  interface   //interface 实现多对一的pat地址转换
//x是编号nat和global后的编号统一才会做地址转换

        pix上关闭nat

        no nat control

nat（接口名）0<源网络><掩码> //较为温和
//nat对应的转换编号是0，那么对应的源网络不转换

      2. asa上的nat

        show xlate //查看asa的转换条目

        1. 配置对象网络，抓取允许被转换的内网源

          object network <name>  //该名字可以是部门名字
subnet <内网ip地址> <子网掩码>

        2. 创建对象组，把所有的对象网络融合在一起

          object-group network <group-name>
object-network object <对象网络名字>

        3. 配置转换语句，将对象组和转换接口关联

          nat（inside,outside） source dynamic <obj-g-name> interface
//inside/outside是接口名字

    5. 防火墙不允许ping的返回流量从outside 接口穿越防火墙

      防火墙上配置acl，放行穿越防火强的流量（从低到高）

      access-list xxx extended permit icmp any any echo-reply
access-group xxx in/out interface outside(套用acl的接口)

    6. 防火墙上发布内网的服务被公网访问

      1. asa：

        object network <name>
host <内网地址>  //服务器所在的真实的内网地址
nat(inside,outside) static <外网地址\interface> service tcp <内port> <外port>

        !!注意：防火墙上如果做服务发布，那么最终一定要用acl允许流量穿越到内网

        access-list <name> extended permit tcp any host 内网服务器地址 eq 内网服务器端口

        防火墙的服务发布是允许内网什么人呢访问，所以acl是放行到内网数据

        ！！终极补充：防火墙流量穿行规则：

        1. 任意大于1安全等级的流量穿越防火强从安全等级0出去都要nat

        2. 但是非0之间通讯，未必需要nat，但是如果安全等级一样，默认不允许通讯，除非录入如下命令：

          same-security-traffic permit inter-interface

防火墙古装倒换：fail over

防火墙的故障倒换实际上就是防火墙的热备份

  1. 故障倒换的模式

    1. A/S：active/standby

      两台防火墙只有active状态负责处理数据，standby只负责监听

    2. A/A:active/active

      两台防火墙同时处于active状态，同时处理数据，且互为备份

  2. 故障倒换的术语

    1. 设备的角色和状态

      主primary，从standby

      故障倒换里，所有主设备都会复制给从设备。主设备通常是active，从设备通常是standby

    2. 配置同步链路及状态同步链路

      配置同步链路，是负责配置同步文件，同步从设备的

      状态同步链路，是负责传递nat信息，同步状态给从设备

    3. 物理接口下的主ip地址和从ip地址

      物理接口下主ip地址是当作主ip信息反送出去的，。两台防火墙都可以配置一样的主地址和从地址，但是只有active状态的设备被认为拥有主地址，另外一边就是从。（最多两人一组）

    4. 故障倒换的方式

      1. 基于线缆的：防火墙有特殊的接口，利用特殊的线缆连接，完成配置和状态的同步
      2. 基于lan的：防火墙消耗一个正常的接口，完成配置和状态的同步。
  • 注意：如果需要使用故障倒换，需要激活码激活

虚拟防火墙

  1. 防火墙的模式

    1. single单模式 默认模式

    2. multiple多模式 支持虚拟防火墙

      show mode

  2. context 内容

    一个context就是一个独立的配置容器，关联接口，关联路径，关联配置，关联保存路径，防火墙系统就会同时运行多个context等于只想能够多个策略。

  • 防火墙默认切换到到模式后，只支持2个虚拟防火墙，需要序列号激活，拥有多个虚拟防火墙

    防火墙上默认有一个叫admin的context用于创建其他context

  1. 虚拟防火墙及故障倒换结合

    同一台防火墙在不同的虚拟防火墙下扮演不同的故障倒换角色，虚拟防火墙不支持动态路由协议，不支持vpn。

vpn

虚拟的专有网络，
可以对数据进行加密和完整性保护，也可以不加密，不保护

  1. vpn分类

    1. osi7层
      1. 应用层 ：app 开源vpn（openvpn）
      2. 会话层，表示层 ：没有vpn
      3. 传输层 ：ssl vpn（安全 socket（套接字/端口））
      4. 网络层 ：ipsec vpn（运用度很广）mpls vpn（2.5层）
      5. 数据链路层 :pptp（点到点隧道协议）,l2tp（二层隧道协议）—-逐渐被淘汰（用于保护操作系统数据传输过程）

    2. 应用

      1. L2L vpn；site to site vpn（局域网到局域网，站点到站点） ：

        ipsec vpn—->DM vpn(基于ipsec，为了取代它，思科私有)【ccie考试必考】

        get vpn （通用的 封装加密 隧道）对mpls vpn的安全加固（思科私有）

      2. remote vpn

        ssl vpn（保护邮件）

        pptp，l2tp ez-vpn（思科私有）—->anyconnect vpn（最新）能适配终端更多（ios/Android…）

    3. 明文数据挑战
      1. 对等体认证
      2. 完整性
      3. 加密
      4. 抵赖—-数字签名

    4. vpn组建

      1. key 密钥

        1. 认证的过程中 key 就是口令
        2. 计算过程中 key是工具，变量

        3. 加密算法key

          1. 对称key ：加密解密同一把key，优点：快，高效，加密后的数据以及不会膨胀太多

          2. 非对称key ：

            1. 公key

            2. 私key

              拿到公私钥互相算不出对方，但可以互相解密

            3. 一般情况下，加密公私钥如何分配

              公钥互相交换，公钥加密数据，私钥解密

            4. 数字签名

          对称密钥加密算法保护数据，非对称密钥加密算法保护过程

    5. 对称，非对称算法

      1. 对称的 ：des；3des；aes
      2. 非对称 ：dsa；rsa

    6. 数据完整性保护算法

      1. md5（消息摘要算法）算完128bit（描述一个算法的强壮程度是bit数）
      2. sha（算完256bit，要去代md5）

    7. key的产生

      dh算法 ，两个随机数算出相同的数，且不可逆（魔方算法）

    8. 启源认证/对等体认证

      1. 预共享口令认证
      2. 数字证书 rsa/signature
      3. 加密随机数（淘汰，傻逼）

    9. lsakmp与ike2个阶段

      1. lsakmp互联网安全关联密钥管理协议
        1. sa安全关联—-连接和过程已经被保护了

      2. ike互联网密钥交换（被包含在isakmp）

        a----------------b
1. ike阶段一
    1. （1.2）互发vpn保护策略（方式一样）
    2. （3.4）互发认证消息
    3. （5.6）互发保护vpn的key
2. ike阶段二
    1. （7.8.9）利用阶段一的key算出保护数据的key，确认加密用户数据的算法和完整性的算法

    10. vpn头部

      1. 隧道模式 tunnel mode
      2. 传输模式 transport mode
      3. ah，esp
        1. ah认证头：加密数据只保证认证和完整性
        2. ESP：封装安全静荷（静荷就是中心数据）：加密数据保证认证和完整性

ipsec vpn的配置

  1. 确认路由器ios支持ipsec vpn

    show version //查看版本
K9的含义代表支持128bit以上的加密算法

  2. 完成ipsec阶段一保护策略的设置

    crypto isakmp policy <策略编号>
authentication pre-share //开启预共享口令认证
encrption <des/3des/aes> //设置加密vpn数据流量方式
hash <md5/sha> //设置vpn流量完整性保护方式
group <1/2/5>  //设置dh组算法级别

  3. 设置对等体地址，并设置预共享口令

    crypto iskmp key <口令> address <对端vpn网关公网地址>

  4. 设置ike阶段二 ，用户数据保护方式和数据封装头部方式和vpn方式

    crypto ipsec transet <转换集名称> esp-<des/3des/aes> esp-<md5/sha>-hmac
mode <tunnel/transport>
  5. 编写扩展acl，利用acl匹配需要被vpn保护的流量

  6. 配置加密映射，将对等体，感兴趣流和阶段二用户保护方式结合

    crypto map <机密映射名字> <加密映射编号> ipsec-isakmp
set peer <对端vpn网关公网地址>
set transet <转换集名字>
match address <acl编号>

  7. 加密映射套用在接口上

    crypto map 只对接口发送的流量生效

    interface <intf-id>
crypto map <加密映射名字>
  8. 查看
    show crypto engine connection active  //查看vpn是否机密解密流量
show access-list  //查看acl有没有被击中

ipsec vpn与nat同时存在的问题

nat会先于vpn抓取到流量，所以nat的转换工作可能会导致vpn无法识别该被加密的流量

  1. nat bypass ：nat绕行

    改acl，使通过vpn到达对端的流量不走nat

ipsec vpn实施方式

  1. 隧道ipsec vpn

    先建立gre隧道，利用隧道完成路由可达，再利用ipsec vpn对隧道保护

  2. 设备出口直接使用ipsec vpn

    设备出口流量会被ipsec vpn识别，判断是否需要加密保护

ipsec vpn的缺陷

  1. 对端设备必须公网地址固定
  2. 如果需要全互联IPSEC VPN保护，配置特别复杂
  3. 无法实现分支之间无需IPSEC VPN配置就能建立IPSEC VPN连接
